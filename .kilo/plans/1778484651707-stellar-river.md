# HTTP 数据源 & 级联加载 - 实现计划

## 一、JSON 配置示例 (14-related-fields.jsonc)

### API 响应结构分析

**GET /users** 返回数组，每个元素包含 `id`, `name`, `email` 等字段。
- 映射：`label` → `name`, `value` → `id`

**GET /posts?userId=1** 返回数组，每个元素包含 `id`, `title`, `body` 等字段。
- 映射：`label` → `title`, `value` → `id`

## 二、实现计划

### Step 1: 扩展类型定义

**文件**: `src/webview/views/formConfigTypes.ts`

新增以下类型接口：

```typescript
// HTTP 数据源配置
export interface DataSourceHttpConfig {
  url: string;
  method?: 'GET' | 'POST' | 'PUT' | 'DELETE';
  headers?: Record<string, string>;
  query?: Record<string, unknown>;
  body?: unknown;
  timeoutMs?: number;
}

// 响应数据转换规则
export interface DataSourceTransform {
  path?: string;              // JSONPath，从响应体提取数组
  labelField?: string;        // 默认 "label"
  valueField?: string;        // 默认 "value"
  disabledField?: string;
  childrenField?: string;     // 树形 children 字段映射
}

// 缓存配置
export interface DataSourceCacheConfig {
  ttl: number;
  key?: string;
}

// 搜索配置
export interface DataSourceSearchConfig {
  debounce?: number;
  minLength?: number;
}

// 数据源完整配置
export interface FormItemDataSource {
  http: DataSourceHttpConfig;
  transform?: DataSourceTransform;
  fallback?: Array<{ label: string; value: unknown; disabled?: boolean }>;
  cache?: DataSourceCacheConfig;
  search?: DataSourceSearchConfig;
  watch?: string[];
  condition?: string;
  clearOnWatchChange?: boolean;
}
```

修改 `FormConfigItem` 接口，新增 `dataSource` 字段：

```typescript
export interface FormConfigItem {
  // ... 现有字段
  dataSource?: FormItemDataSource;  // 新增
}
```

### Step 2: 创建 useDataSource Hook

**新文件**: `src/webview/hooks/useDataSource.ts`

核心功能：
- 接收 `dataSource` 配置 + `formData` + `interpolate` 函数
- 管理 `options`, `loading`, `error` 状态
- 实现缓存机制（内存 Map + TTL）
- 监听 `watch` 字段变化，自动重新请求
- 评估 `condition` 表达式，决定是否发起请求
- 调用 `httpRequest` 发起请求
- 通过 `transform` 映射响应数据为标准 options 格式

关键逻辑：
```typescript
export function useDataSource(
  dataSource: FormItemDataSource | undefined,
  formData: Record<string, unknown>,
  httpRequest: (req: HttpRequestOptions) => Promise<HttpResponseResult>,
  onChange: (keyName: string, value: unknown) => void,
): { options: Array<{ label: string; value: unknown }>; loading: boolean }
```

### Step 3: 修改 AntdFormView 集成数据源

**文件**: `src/webview/views/AntdFormView.tsx`

修改点：

1. **AntdFormItem 组件**：
   - 当 `item.dataSource` 存在时，调用 `useDataSource` hook
   - 将 hook 返回的 `options` 与静态 `item.options` 合并（dataSource 优先）
   - 传递合并后的 options 给组件

2. **AntdFormView 组件**：
   - 在 `handleValuesChange` 中，检测被 `watch` 的字段变化时，触发依赖项重新加载
   - 当 `clearOnWatchChange: true` 且 watch 字段变化时，清空下游字段值

### Step 4: 扩展 interpolate 支持

**文件**: `src/webview/views/SubmitBar.tsx` 或提取为公共工具

当前 `interpolate` 函数已支持 `{{path}}` 语法，需确认：
- 能正确处理 `{{selectA}}` → 从 formData 取值
- 支持条件表达式如 `{{selectA}} != null && {{selectA}} != ''`

如果需要更复杂的条件评估，考虑使用 `Function` 构造器或轻量表达式解析器。

## 四、技术要点

1. **HTTP 请求复用**: 直接使用 `useVSCodeBridge().httpRequest`，保持与 SubmitBar 一致的请求处理
2. **模板插值复用**: 复用 `interpolate` 函数处理 `{{selectA}}` 等占位符
3. **向后兼容**: `options` 字段保持不变，`dataSource` 是可选新增，两者共存时 dataSource 优先
4. **错误处理**: 请求失败时显示 fallback，不阻断表单
5. **缓存策略**: 相同 URL + query 组合生成缓存 key，TTL 控制有效期
6. **级联清空**: `clearOnWatchChange` 确保父级变化时子级不会保留无效选中值
